                                                    1 МЕТА РОБОТИ

Метою даної лабораторної роботи є розробка документа Vision and Scope для програмної системи моніторингу умов робочого середовища корпоративних приміщеннях.
2 ХІД РОБОТИ
У процесі виконання роботи були визначені основні вимоги до програмної системи моніторингу умов робочих місць, що включає використання IoT-сенсорів для моніторингу параметрів, таких як температура, вологість та рівень CO₂. Розроблена система дозволяє співробітникам легко отримувати дані про умови в офісі через мобільний додаток, а також отримувати сповіщення про зміни в робочому середовищі. Було також здійснено аналіз існуючих рішень у цій сфері, що дозволило виявити обмеження і визначити ключові напрямки для вдосконалення. Також було розроблено документ Vision and Scope, що деталізує концепцію системи та її основні функціональні блоки.

3 ВИСНОВКИ
Розроблена система моніторингу умов робочого середовища відповідає вимогам, забезпечуючи автоматизований процес збору і аналізу даних, а також зручне управління і сповіщення про зміни в умовах роботи. Впровадження IoT-сенсорів для моніторингу фізичних умов на робочих місцях та мобільного додатка для комунікації співробітників дозволяє забезпечити ефективність роботи і безпеку користувачів. Аналіз поточних рішень на ринку дозволив визначити ключові напрямки для поліпшення якості обслуговування і надання послуг в системах моніторингу.

1.	BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)


1.1 Background (Передумови)



У сучасному корпоративному середовищі важливе значення мають комфорт і безпека робочого простору. Відсутність належного моніторингу умов у приміщеннях може призводити до зниження продуктивності співробітників, погіршення їх здоров'я та порушення трудових процесів. Температура, вологість та рівень CO₂ є ключовими показниками, які необхідно контролювати, особливо в офісах, лабораторіях та на полігонах.
Існує також потреба у швидкому інформуванні співробітників про зміни в умовах робочого середовища та забезпеченні ефективної взаємодії між ними. Застарілі методи обміну інформацією, такі як паперові повідомлення чи телефонні дзвінки, більше не відповідають сучасним вимогам до швидкості та зручності комунікації.
Програмна система моніторингу умов робочого середовища корпоративних приміщень вирішує цю проблему. Вона дозволяє автоматизувати процес збору, аналізу та візуалізації даних про стан робочих просторів за допомогою IoT-сенсорів, мобільного додатка та веб-інтерфейсу. Система також забезпечує швидку комунікацію між співробітниками та оперативне реагування на зміни умов.


1.2	     (Business Opportunity) Бізнес можливості


WorkEnviro: Smart Workplace Monitoring System пропонує інноваційне рішення для моніторингу умов робочого середовища в корпоративних приміщеннях. Система використовує IoT-сенсори для вимірювання температури, вологості, рівня CO₂ та інших показників, що дозволяє забезпечити оптимальні умови для роботи. Крім того, мобільний додаток забезпечує реальний моніторинг і сповіщення про зміни умов, даючи змогу співробітникам оперативно реагувати на зміни середовища.
Переваги системи: автоматизований збір даних у реальному часі, інтеграція з мобільними пристроями для оперативного інформування співробітників, зручна комунікація через месенджер для обговорення умов та дій. Система допомагає знизити ризики, пов'язані з поганими умовами праці, підвищуючи продуктивність та комфорт співробітників.
Недоліки: високі початкові витрати на впровадження IoT-сенсорів та підтримку системи, необхідність стабільного інтернет-з'єднання для належної роботи.

1.3	 Способи монетизації


• Підписка на систему моніторингу: Основне джерело доходу — підписка на використання програмної системи моніторингу умов робочого середовища. Корпорації платять за доступ до платформи та її функціоналу для відстеження умов у своїх приміщеннях.
• Додаткові послуги: Можливість надання додаткових послуг, таких як аналітика для покращення умов роботи, налаштування IoT-сенсорів, інтеграція з іншими корпоративними системами (наприклад, управління будівлею), а також надання преміум-сповіщень. Ці послуги можуть надаватись за додаткову плату, що збільшує загальний дохід.
 
2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)


2.1 Vision Statement (Окреслення концепції)


Програмна система моніторингу умов робочого середовища забезпечить компаніям ефективний спосіб відстеження умов у їхніх офісах, лабораторіях та на полігонах. Система буде використовувати IoT сенсори для моніторингу температури, вологості та рівня CO₂, щоб гарантувати комфортні та безпечні умови для працівників. Мобільний додаток дозволить користувачам швидко отримувати сповіщення про зміни в умовах робочого середовища, а також здійснювати комунікацію з колегами в реальному часі. Платформа стане інтегрованим рішенням для контролю за станом приміщень, надаючи прозорі дані та зручні інструменти для управління.


2.2 Major Features (Головна функціональність)


Основні функції програмної системи моніторингу умов робочого середовища:
•	MF 1) Моніторинг умов у реальному часі (температура, вологість, рівень CO₂) в офісах, лабораторіях та на полігонах.
•	MF 2) Сповіщення користувачів про зміни в умовах середовища через мобільний додаток.
•	MF 3) Відображення історії змін умов у приміщеннях та надання аналітики для покращення середовища.
•	MF 4) Взаємодія між співробітниками через вбудований месенджер для обговорення змін та координації дій.
•	MF 5) Реєстрація користувачів для отримання доступу до моніторингу та налаштування персональних параметрів.
•	MF 6) Інтерфейс для перегляду зайнятості робочих місць та аналізу тенденцій у використанні приміщень.

 
3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)


3.1 Scope of Initial Release (Рамки первинного випуску)


Перелік функціональності для серверної та IoT частини програмної системи моніторингу умов робочого середовища:
•	MF 2-1) Збір і зберігання даних з IoT сенсорів про температуру, вологість та рівень CO₂ у робочих приміщеннях.
•	MF 2-2) Оповіщення користувачів через мобільний додаток про зміни в умовах робочого середовища.
•	MF 3-1) Автоматизація сповіщення користувачів про небезпечні або незручні умови (висока температура, низька вологість тощо).
•	MF 4-1) Реєстрація користувачів для доступу до системи моніторингу та отримання сповіщень.
•	MF 5-1) Надання доступу до історії змін умов середовища в компанії та перегляд поточної інформації через веб чи мобільний додаток.


3.2 Scope of Subsequent Releases (Рамки наступних випусків)


Перелік функціональності для веб та мобільних клієнтів програмної системи моніторингу умов робочого середовища:
•	MF 2-3) Візуалізація даних у реальному часі через графіки та діаграми для аналізу умов робочих місць.
•	MF 3-2) Надсилання сповіщень через email та інтеграція з корпоративними системами для моніторингу.
•	MF 6-1) Створення звітів та аналітики для оцінки тенденцій змін у робочому середовищі та використання приміщень.
 
4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)


4.3 Operating Environment (Робоче середовище)


Система моніторингу умов робочого середовища буде використовуватися користувачами в різних регіонах та часових зонах. Основне завдання системи — забезпечити доступ до даних про умови робочого середовища (температура, вологість, рівень CO₂) в реальному часі, надаючи можливість переглядати історію змін та отримувати сповіщення про небезпечні або незручні умови для роботи.
Система інтегруватиметься з мобільними додатками, веб-платформою та IoT пристроями для збору даних з робочих приміщень, що потребує стабільного інтернет-з'єднання та високої надійності для точного моніторингу умов. Програма також повинна бути здатна працювати з великою кількістю користувачів одночасно, забезпечуючи безперебійну роботу та надання актуальних сповіщень.


Мови програмування:

•	JavaScript (Node.js, Express): Для розробки серверної частини, що забезпечує ефективну роботу RESTful API та інтеграцію з базою даних.
•	JavaScript (React): Для створення інтерактивного веб-сайту, що забезпечує зручний доступ до даних та сповіщень про умови робочого середовища.
•	Kotlin (Android): Для розробки мобільного додатку
•	
Інфраструктура серверної частини:

•	Express: Веб-фреймворк для створення RESTful API та організації серверної логіки.
•	MongoDB: NoSQL база даних для зберігання даних про умови в приміщеннях та історію змін.
•	Mongoose: ORM для MongoDB, що спрощує роботу з базою даних.

•	MQTT: Протокол для обміну повідомленнями між IoT пристроями та сервером, що дозволяє збирати дані в реальному часі.
•	JWT: Система авторизації на основі токенів для захисту даних користувачів.
•	Docker: Для контейнеризації серверної частини та спрощення процесів розробки та деплойменту.


Інфраструктура фронтенд частини:

•	React: Бібліотека для створення інтерфейсу веб-додатку.

•	Material-UI: Для стилізації інтерфейсу та забезпечення зручності використання.
•	Axios: Для виконання HTTP-запитів до серверу та взаємодії з API.
•	
Інфраструктура мобільного додатку:

•	Kotlin (Android): Для розробки мобільного додатку
